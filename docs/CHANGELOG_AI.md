- **Main page not reflecting Registry Editor saves (registry.json) — fix**
  - **Issue:** After editing in Registry Editor and Save, the main page did not show the new values; it still showed defaults or old content/style.
  - **Root cause:** When registry is loaded from `registry.json` (file), `Index` on mount was calling `refresh()`, `refreshStyleRegistry()`, and `applyStyleRegistry(getMergedStyleRegistry())`. Those overwrote file-sourced state and DOM with **localStorage** (and defaults), so the file’s content/style were replaced before render.
  - **Changes:** (1) **RegistryContentContext:** When `initialContent` is provided (from file), `refresh()` is a no-op so content/contentModifiers are not overwritten by localStorage. Context now exposes `fromFile` so consumers know the source. (2) **StyleRegistryContext:** When `initialStyle` is provided (from file), `refreshStyleRegistry()` is a no-op so registry state is not overwritten by `getMergedStyleRegistry()`. (3) **Index:** When `fromFile` is true, the mount effect skips `applyStyleRegistry(getMergedStyleRegistry())`, `refresh()`, and `refreshStyleRegistry()`, so the main page keeps the registry.json values applied at bootstrap.
  - **Files touched:** `src/contexts/RegistryContentContext.tsx`, `src/contexts/StyleRegistryContext.tsx`, `src/pages/Index.tsx`, `docs/CHANGELOG_AI.md`.
  - **Verification:** Build succeeds. No FAILURE_LOG. Manual: edit in Registry Editor → Save → reload → main page shows saved content/style.
